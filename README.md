# ИИ Сисадмин Академия

**AI-Powered Educational Platform for SysAdmins & DevOps**

Современная образовательная платформа с ИИ-наставником для системных администраторов, DevOps-инженеров и специалистов по инфраструктуре. Актуально на июнь 2026 года.

## 🚀 Возможности

- **15 категорий**: Linux, Сети, Bash, Безопасность, Docker, Kubernetes, Terraform, Ansible, AWS, Azure, GCP, DevOps, Мониторинг, Базы данных, Веб-серверы
- **ИИ-наставник**: чат с AI-ассистентом, специализирующимся на системном администрировании
- **Markdown-форматирование контента**: ответы, AI-объяснения и сообщения чата рендерятся с разделением на блоки — заголовки с акцентной полосой, цветные буллеты списков, блоки кода с подсветкой и кнопкой копирования, цитаты-советы, таблицы с зеброй
- **OpenRouter интеграция**: бесплатные LLM-модели с возможностью добавления своих
- **Управление токенами**: добавление, проверка, хранение API-ключей
- **Проверка моделей**: тестирование доступности и задержки каждой модели
- **Режим обучения**: изучение вопросов по категориям с AI-объяснениями
- **Режим собеседования**: тренировка перед техническими интервью
- **Система прогресса**: XP, уровни (Новичок → Архитектор)
- **Тёмная/светлая тема**: адаптивный дизайн для всех экранов

## 🎨 Форматирование контента

Весь текстовый контент (ответы, AI-объяснения, сообщения чата, обратная связь ИИ-наставника) рендерится через переиспользуемый компонент `MarkdownContent` (`src/components/ui/markdown.tsx`):

- **Заголовки** (`#`, `##`, `###`) — с акцентной цветной полосой слева для визуального разделения секций
- **Маркированные списки** (`-`) — с цветными буллетами фирменного цвета
- **Нумерованные списки** (`1.`) — с номерами в круглых бейджах
- **Блоки кода** (``` ``` ) — моноширинный шрифт, тёмный фон, скруглённые углы, кнопка «Копировать», горизонтальная прокрутка
- **Инлайн-код** (`` `code` ``) — подсветка фоном
- **Цитаты** (`>`) — с цветной полосой слева, используются для советов
- **Таблицы** (GFM) — со строками-зеброй
- **Жирный/курсив** — стандартные стили

Три размера: `sm` (для чата), `md` (для карточек), `lg` (для основного контента).

## 📦 Установка

```bash
# Клонирование
git clone https://github.com/ai-sysadmin-academy/ai-sysadmin-academy.git
cd ai-sysadmin-academy

# Установка зависимостей
npm install

# Инициализация базы данных
npx prisma db push
npx prisma generate

# Заполнение тестовыми данными
npx tsx scripts/seed.ts

# Запуск в dev-режиме
npm run dev
```

## 🔑 API-токен OpenRouter

Для работы ИИ-наставника:

1. Перейдите на [openrouter.ai/keys](https://openrouter.ai/keys)
2. Зарегистрируйтесь и создайте API-ключ
3. Добавьте ключ в приложении через кнопку «Добавить свой токен»

Бесплатные модели работают и без ключа, но с ограничениями по скорости.

## 🔬 LLM Security Lab

Отдельный проект в разделе веб-безопасности — исследование Prompt Injection атак на LLM-приложения.

**Доступ**: `/llm-security` (ссылка в header приложения)

### Возможности
- **Уязвимый режим** — все защитные правила отключены, можно эксплуатировать уязвимости
- **Защищённый режим** — 5 security rules (input filter, prompt isolation, least privilege, human-in-the-loop, output validation)
- **7 готовых payloads** для тестирования атак
- **6 LLM-функций** с разными уровнями привилегий (read/write/delete)
- **Audit Log** — логирование всех вызовов функций

### Демонстрируемые атаки
1. **Прямая инъекция** (Jailbreak) — «Ignore previous instructions»
2. **Утечка системного промпта** — System Prompt Leakage
3. **Косвенная инъекция** — скрытый payload в HTML-комментарии
4. **Эксплуатация API** — delete_account, change_email без подтверждения
5. **Утечка данных** — раскрытие API-ключей

### Документация
- [`docs/llm-security/PENTEST_REPORT.md`](docs/llm-security/PENTEST_REPORT.md) — отчёт о тестировании на проникновение
- [`docs/llm-security/SECURE_DEV_GUIDE.md`](docs/llm-security/SECURE_DEV_GUIDE.md) — руководство разработчика по безопасности LLM

Основано на [PortSwigger Web Security Academy](https://portswigger.net/web-security/llm-attacks) и OWASP Top 10 for LLM.
